Verdict, for the "is this worth adding" conversation

This is the kind of moment PixiJS is actually good for — hundreds of live particles, a real wipe/ mask reveal, additive glow — all things CSS can only approximate. It would not replace the rest of the UI (nav, cards, forms stay DOM/Nuxt UI), just moments like this one and the recruit-vetting animation.
  • Gain: particle density and glow quality CSS can't match without a lot of hacks; a real progressive reveal (mask-based) instead of the CSS version's flash-cut.
  • Cost: new dependency, canvas needs client-only mounting in Nuxt (no SSR), and it's a different mental model (imperative scene graph + ticker) from the rest of the Vue-templated app — worth isolating behind one composable/component so it doesn't leak into everything else.
  • Recommendation: scope it to 2–3 cinematic moments (crate open, recruit vetting, maybe the rarity stamp burst) and leave everything else exactly as designed. Try it in the real app on one screen before deciding to commit further.
